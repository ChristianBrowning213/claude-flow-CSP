# Memory Server

Purpose:
- Treat memory as a structured server subsystem rather than chat transcript storage.

Tools:
- `memory.put`
- `memory.get`
- `memory.search`
- `memory.link`
- `memory.consolidate`

Store types:
- `ToolShape`
- `PlanMemory`
- `ConstraintMemory`
- `RunMemory`
- `EvidenceBundle`

Scoping:
- `project`
- `repo`
- `run_id`
- `experiment_id`
